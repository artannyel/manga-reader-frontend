# Architecture & Planning Skill (Frontend)

This skill is used to design frontend components and generate SDD documentation.

## Guidelines

1. **Routing & Navigation**:
   - Map out GoRouter routes, query parameters, path variables, and redirects (e.g. redirecting to offline/network-error pages).

2. **State & Providers**:
   - Design Riverpod providers, StateNotifier/Notifier states, and data flows.
   - Outline local storage (Isar/Hive) schema entities and attributes.

3. **Layer Separation**:
   - Ensure clear mapping between Data Models, Domain Entities, and Presentation states.

4. **Planning**:
   - Create phased implementation plans and a task checklist file.
